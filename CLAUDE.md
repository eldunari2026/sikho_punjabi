# Sikho Punjabi — Claude Code Instructions

## Session Start

At the start of every new conversation, before doing any work, ask:

> "Working mode for this session: **(1) Claude-only** or **(2) Codex-assisted**?"

Wait for the user's answer before proceeding. Default to Claude-only if no answer is given.

---

## Mode 1: Claude-only

Claude writes all code, plans, reviews, and runs commands. No Codex invocation.

---

## Mode 2: Codex-assisted

### Division of labor
- **Claude does:** planning, architecture, spec writing, code review, edits to existing code, running commands, debugging, orchestration
- **Codex does:** new module/file implementations — any self-contained coding task requiring new code where inputs/outputs and edge cases are well-defined

### When to use Codex vs Claude directly
- **Use Codex:** new files or modules, large function implementations, pipeline components, data transforms
- **Use Claude directly:** bug fixes, small edits, glue code, anything requiring reading across many existing files

### How to invoke Codex
1. Write a spec first: function signatures, inputs/outputs, edge cases, file location
2. Run `codex "<spec>"` via Bash
3. Review the diff and edit as needed before accepting

### Spec discipline
Always produce a written spec before calling Codex. The spec is the review contract — if output doesn't match the spec, that's the failure point. Never send vague prompts to Codex.

---

## Project Rules (both modes)

### Azure OpenAI
- All LLM calls in enrichment scripts use Azure OpenAI (`gpt-5.4`), not Claude
- Use `AzureOpenAI` client from `openai`, not the standard `OpenAI` client
- Claude orchestrates; Azure OpenAI executes NLP and coding tasks in the pipeline

### Romanization
- Two-field schema only: `readable` (LLM-generated) + `tonal` (rule-based)
- ISO 15919 is dropped — do not reintroduce it
- Do not add a third romanization field

### Tonal algorithm
- Tone marks are generated by the rule-based algorithm in `tonal.py`, not by an LLM
- Do not replace or bypass it with an LLM call
